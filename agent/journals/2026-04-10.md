Reviewed the DynService->MutDynService migration in flow-venues and the new standalone tcp_server.rs FlowSessionManager/FlowSessionDispatcher. Overall looks correct. Key findings:
- MutDynService drops the Send requirement, enabling RefCellLock (Rc<RefCell>) for standalone's single-threaded worker
- RefCellLock is only created inside the worker thread (via the FnOnce closure), never crosses thread boundaries — safe
- Session cleanup covers all error paths (WS receive error, sequencing error, fatal send/poll errors, WS send error)
- The shared Rc<RefCell<HashMap>> between FlowSessionManager and FlowSessionDispatcher is safe because MutServiceChain calls them sequentially, and FlowSessionDispatcher explicitly drops the borrow before re-borrowing for GC
