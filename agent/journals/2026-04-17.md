**Marvel system**: Full-stack trading platform. C++ for critical path (MD gateway, order gateway, trading app), Python/Go/React for non-critical, MongoDB/CassandraDB/QuasarDB for storage. Config-driven; can run as monolith or distributed. Equinix NY4+LD5, AWS US/EU/Asia.

**vs Talos**: Strong overlap — streaming RFQ, per-client markups, credit/leverage controls, LP monitoring, FIX 4.4 + WS + REST APIs, 50+ MD feeds, 25+ order gateways. Voice trade / RFQ chat booking is actual production use. Vision OMS (internal) + Vision Trader (client portal) = Studio-equivalent. Credible Talos replacement if tech holds.

**vs Haruko**: Partial. Marvel has loans/swaps, automated margin calls, client liquidation, margin position tracking. But no description of full loan lifecycle (amend/recall/settlement), no vol surface/greeks, no front office risk view. Probably reduces Haruko dependency on execution side; unlikely to eliminate it for risk/front office day 1.

**Integration concerns**: Proprietary message bus (not Kafka natively — Kafka only listed as output log option); no ledger replacement for BK; C++/Python/heterogeneous DBs vs Rust-first CS stack; bankruptcy state means team may be gone.

**Strengths**: 400+ spot crypto + thousands of derivatives across 10+ venues; options/perps/CFDs/structured products all covered; tick data to 2017; DAG-based configurable strategy building blocks; pre-written market making + liquidity aggregation + risk hedging strategies.
